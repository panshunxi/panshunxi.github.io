pig4cloud项目部署

将pig4cloud部署到本地vm虚拟机的Linux服务器上，完整的部署流程。

一、准备工作

提前准备工作：

  1. VMware虚拟机安装Cent OS7系统（网上有很多，略）
  2. 把前后端代码从官网克隆到本地，并且能够运行（官网有详细的在本地运行的步骤，略）

开启此虚拟机时，遇到“无法打开内核设备“\\.\VMCIDev\VMX”: 操作成功完成。是否在安装 VMware Workstation 后重新引导?”

解决办法：

  1. 找到该虚拟机的存放位置

  2. 用记事本打开虚拟机名称.vmx

  3. 找到文件里面vmci0.present="TRUE"，改成vmci0.present="FALSE"（记得保存）

  4. 重启虚拟机

网卡设置

  1. 如何查看服务器IP地址ip addr

  1. 配置网卡初始化

    a. 第一步：进入路径cd /etc/sysconfig/network-scripts

    b. 第二步：编辑网卡vi ifcfg-ens33

    ​ 进入文件后执行如下操作：

    ​ ①按i键，进入编辑

    ​ ②按↑ ↓移动光标，将ONBOOT=no改为ONBOOT=yes

    ​ ③按Esc键，退出

    ​ ④输入:wq，再按Enter回车键保存并退出

  2. 关机重启虚拟机

  3. 如果该Linux服务器部署项目，目前设置为是动态获取IP地址，也就意味着，IP地址可能会发生改变，所以最好把IP地址设置为静态的

    设置静态IP，修改配置文件vi /etc/sysconfig/network-scripts/ifcfg-ens33

    内容有IP、子网掩码、网关地址、DNS服务器

  4. 关机重启，或者执行重启网络服务的指令systemctl restart network

远程连接服务器（这里使用的是FinalShell软件）

三、常见问题

1. 验证码图片不加载

验证码图片没有加载，控制台报错GET http://192.168.213.130/api/auth/code/image?randomStr=f3ff0767-69ba-4f99-8486-ca1a56da4cc6 500 (Internal Server Error)

解决办法：后端已经运行，可以查看对应的日志

2. 重启服务器后无法访问

访问http://ip地址:80出现无法访问的情况（重启服务器时再访问就经常遇到）

解决办法：重新启动nginx（nginx掉线了）；以及Redis也需要重新启动；重新禁用SELinux

网关出现问题

解决办法：检查网关是否运行
